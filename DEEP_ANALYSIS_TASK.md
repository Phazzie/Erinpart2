## 🔍 PHASE 1: ULTRA-DEEP CODE ARCHAEOLOGY

### 🎯 Search Methodology: LEAVE NO STONE UNTURNED

**Go beyond surface-level scanning. You must:**

#### A. SYSTEMATIC CODE ANALYSIS
- **Every File Inspection**: Read EVERY `.ts`, `.tsx`, `.js`, `.json`, `.md` file
- **Pattern Detection**: Look for recurring issues, inconsistencies, anti-patterns
- **Cross-Reference Validation**: Check if implementations match documentation
- **Type Safety Audit**: Find any `any` types, unsafe casts, missing type definitions
- **Performance Bottlenecks**: Identify inefficient patterns, memory leaks, unnecessary re-renders

#### B. INTEGRATION & CONSISTENCY ANALYSIS  
- **API Boundary Verification**: All server actions, database queries, real-time subscriptions
- **State Management Audit**: Hook dependencies, state synchronization, race conditions
- **UI/UX Inconsistencies**: Design system violations, accessibility gaps, mobile issues
- **Error Handling Gaps**: Missing try-catch blocks, unhandled promises, silent failures
- **Security Vulnerabilities**: RLS policy gaps, data exposure, injection risks

#### C. HIDDEN PROBLEMS DETECTION
- **Edge Cases**: What happens with malformed data, network failures, concurrent users?
- **Resource Management**: Memory leaks, connection pooling, cleanup on unmount
- **Browser Compatibility**: Modern JS features without polyfills, CSS gaps
- **Scalability Issues**: N+1 queries, inefficient pagination, large dataset handling
- **Configuration Drift**: Environment variables, build settings, deployment configs

#### D. ARCHITECTURAL CONCERNS
- **Code Organization**: Module boundaries, circular dependencies, coupling issues
- **Testing Gaps**: Untested critical paths, missing integration tests, flaky tests
- **Documentation Gaps**: Missing JSDoc, outdated READMEs, undocumented APIs
- **Deployment Risks**: Missing health checks, inadequate monitoring, rollback strategies
